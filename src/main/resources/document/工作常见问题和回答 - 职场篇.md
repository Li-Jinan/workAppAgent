# 工作常见问题和回答 - 职场篇
#### 工作任务太多，如何安排优先级？
先把任务分成紧急重要、重要不紧急、紧急不重要、不紧急不重要四类。紧急重要的任务立即处理，重要不紧急的任务安排专注时间推进，紧急不重要的任务尽量沟通截止时间或拆给合适的人，不紧急不重要的任务延后或取消。每天开始前列出 3 个必须完成的关键结果，避免被消息和临时需求牵着走。
推荐方法：[《每日三件事工作法》](https://www.codefather.cn)，适合减少忙乱感并提高交付稳定性。

#### 和同事沟通需求总是反复，怎么办？
沟通前先写清楚背景、目标、范围、验收标准和截止时间。会议中确认每个人对“完成”的理解是否一致，会议后用文字同步结论，尤其是接口字段、页面状态、异常逻辑和时间节点。需求变更时记录变更原因、影响范围和新的排期，不要只在口头上达成模糊共识。
推荐模板：[《需求沟通确认单》](https://www.codefather.cn)，可以降低返工和扯皮概率。

#### 如何向上级汇报工作进展？
汇报不要只说“我做了什么”，要说“目标是什么、当前进度、遇到的问题、下一步计划、需要什么支持”。如果有风险，提前暴露并给出备选方案。比如“接口开发完成 80%，联调卡在权限字段不一致，今天下午会和前端确认字段方案；如果对方排期冲突，我会先用 mock 数据保证页面联调推进”。
推荐结构：[《进展-风险-计划汇报法》](https://www.codefather.cn)，适合周报、日报和项目同步。

#### 工作中遇到冲突，如何处理更专业？
先把人和问题分开，不要急着判断对错。明确冲突背后的目标差异，例如质量、速度、成本、用户体验或资源分配。沟通时描述事实和影响，再提出可选方案。比如“如果今天强行上线，测试覆盖不足，可能影响核心流程；方案一是只发低风险功能，方案二是延期一天补测”。用方案讨论替代情绪对抗。
推荐练习：[《职场冲突复盘表》](https://www.codefather.cn)，帮助你把冲突转化成流程改进。

#### 如何提升工作效率而不是单纯加班？
先找出低效来源：需求不清、频繁切换、重复劳动、工具不熟或缺少反馈。对重复任务做脚本化或模板化，对复杂任务做拆分，对沟通类任务提前写清问题列表，对深度工作安排固定时间块。效率提升的核心不是做得更快，而是减少无效返工和无意义切换。
推荐工具：[《个人工作流优化清单》](https://www.codefather.cn)，适合每周复盘自己的时间使用方式。
