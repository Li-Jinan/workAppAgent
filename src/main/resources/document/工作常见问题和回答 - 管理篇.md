# 工作常见问题和回答 - 管理篇
#### 如何把团队目标拆成可执行任务？
先明确目标的业务结果，再拆成阶段里程碑、关键任务、负责人和验收标准。每个任务都要能回答四个问题：谁负责、什么时候完成、交付物是什么、怎样算完成。对于跨多人协作的任务，要提前标出依赖关系和风险点。目标拆解不是把事情切碎，而是让团队知道优先级和交付边界。
推荐模板：[《目标拆解与责任分配表》](https://www.codefather.cn)，适合项目启动会和迭代计划。

#### 团队成员执行力不足，管理者该怎么做？
先判断是意愿问题、能力问题、资源问题还是目标不清。意愿问题需要沟通动机和激励；能力问题需要培训、示范或结对；资源问题需要协调时间、人手或工具；目标不清则要重新定义任务和验收标准。不要一上来只评价“执行力差”，而是定位具体阻塞点。
推荐方法：[《执行问题四象限诊断》](https://www.codefather.cn)，帮助管理者更准确地给支持。

#### 如何做有效的一对一沟通？
一对一沟通要固定频率，并留给成员表达问题和目标的空间。可以按“最近进展、遇到困难、需要支持、成长目标、反馈建议”五个部分展开。管理者要少做即时评判，多问开放问题，例如“你觉得目前最大的阻碍是什么”“如果资源允许，你会怎么推进”。沟通后记录共识和行动项。
推荐模板：[《1:1 沟通记录表》](https://www.codefather.cn)，适合团队管理和新人培养。

#### 项目延期时如何对外同步？
不要等到最后一天才说延期。发现风险时，尽早说明当前进度、延期原因、影响范围和新的交付计划。同步时给出可选方案，例如缩小范围按期交付、延期完整交付、增加资源压缩时间。这样能让相关方参与取舍，而不是被动接受坏消息。
推荐结构：[《项目风险同步模板》](https://www.codefather.cn)，适合跨部门协作和上线延期场景。

#### 如何建立健康的团队反馈机制？
反馈要及时、具体、基于事实，并指向行为改进。正向反馈说明做得好的行为和产生的影响，改进反馈说明观察到的问题、影响和期待的新行为。避免只说“不错”或“不够主动”。团队层面可以在迭代复盘中讨论流程问题，把反馈从个人指责转为系统优化。
推荐工具：[《SBI 反馈模型》](https://www.codefather.cn)，适合代码评审、绩效沟通和团队复盘。
