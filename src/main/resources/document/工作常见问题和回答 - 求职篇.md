# 工作常见问题和回答 - 求职篇
#### 如何优化简历，让招聘方快速看到匹配度？
简历要围绕目标岗位重写，而不是把所有经历简单堆上去。先拆解岗位 JD 中的核心能力，如 Java 后端、Spring Boot、数据库、AI 应用开发、部署经验等，再把自己的项目经历改成“背景、任务、行动、结果”的结构。每段项目经历都尽量包含技术栈、个人负责模块、难点解决和量化结果。避免泛泛写“熟悉”“了解”，改成“使用某技术完成某功能，解决某问题”。
推荐模板：[《岗位匹配型简历检查清单》](https://www.codefather.cn)，适合投递前逐项确认关键词、项目亮点和表达清晰度。

#### 面试前应该如何准备项目介绍？
准备项目介绍时，建议按 1 分钟、3 分钟、5 分钟三个版本分别演练。1 分钟讲项目价值和你负责的核心模块；3 分钟补充技术选型、架构流程和关键难点；5 分钟展开性能优化、异常处理、部署方案和复盘。面试官追问时，优先回答“为什么这么做”“还有什么替代方案”“线上出问题如何排查”。
推荐模板：[《项目介绍 STAR 表达法》](https://www.codefather.cn)，帮助你把项目经历讲得具体、有逻辑、有技术深度。

#### 没有太多实习经历，如何提升求职竞争力？
可以用完整项目和可展示成果补足经历短板。选择一个明确业务场景，例如 AI 工作助手、文件管理系统、数据看板或个人博客平台，从需求分析、接口设计、数据库设计、前端页面、部署上线到 README 文档都做完整。把代码托管到 GitHub，并在简历中附上在线访问地址、接口文档和项目截图。比起只写学习过哪些技术，一个可运行、可解释、可迭代的项目更容易证明能力。
推荐方向：[《从 0 到 1 打造可面试项目》](https://www.codefather.cn)，适合需要用作品集证明工程能力的求职者。

#### 面试被问到不会的问题怎么办？
先承认自己没有在生产中深入使用过，不要编造经验。随后可以说明自己的理解边界、相关知识和排查思路，例如“这个点我没有完整实践过，但我理解它解决的是某类问题。如果让我落地，我会先查官方文档、做最小 demo、再评估和现有系统的兼容性”。这种回答比沉默或硬猜更稳，也能展示学习能力和工程意识。
推荐练习：[《不会问题的高质量回答框架》](https://www.codefather.cn)，帮助你把未知问题转化成可讨论的问题。

#### 如何判断一个岗位是否适合自己？
可以从技术成长、业务方向、团队节奏、薪资范围和长期职业目标五个维度判断。技术成长看是否能接触真实业务、代码评审、上线流程和问题排查；业务方向看自己是否愿意长期投入；团队节奏看加班强度和协作方式；薪资范围看是否匹配生活成本；长期目标看这个岗位能否帮助你积累下一阶段需要的能力。不要只看岗位名称，要看实际工作内容。
推荐工具：[《Offer 选择评分表》](https://www.codefather.cn)，适合在多个机会之间做理性比较。
